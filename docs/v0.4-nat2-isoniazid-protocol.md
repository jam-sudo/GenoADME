# v0.4 Protocol: NAT2 / Isoniazid Validation (Tier 1 Promotion)

> Research and educational tool only; not FDA-cleared.

This document is the **pre-registered protocol** for the v0.4 NAT2/isoniazid pair activation. The protocol section is committed BEFORE any literature search or validation run is executed; the findings section is appended only after execution.

The pre-registration discipline mirrors [`docs/v0.3-meta-analysis.md`](v0.3-meta-analysis.md) and [`docs/holdout-seed.md`](holdout-seed.md). The v0.3 PATH 3 lesson (see `v0.3-meta-analysis.md` §2.10) — *"verify cited references before using them as a spec anchor"* — is explicitly applied here: reference values are NOT locked at protocol time; only the search protocol that will produce them is locked. Verification of each cited reference (does the paper actually cover the drug/gene of interest? does the abstract actually report the cited numbers?) is part of Phase 2 execution.

The v0.4 milestone description ([GenoADME milestone/2](https://github.com/jam-sudo/GenoADME/milestone/2)) names NAT2/isoniazid as the most likely first candidate because Sisyphus PR #32 (v0.3.2, merged 2026-05-06) added NAT2 to `PHENOTYPE_SCALES` and `reference_man.yaml`, resolving the physiology-side blocker documented in [`docs/limitations.md` §9.4](limitations.md). The remaining work is GenoADME-side: NAT2 marker variant calls + holdout integration + Tier 1 promotion.

## 1. Pre-registered protocol

**Date locked:** 2026-05-12.
**Status:** Locked. Any post-lock change is recorded in §1.8 with reason + timestamp.

### 1.1 Tier assignment

**NAT2/isoniazid is promoted to Tier 1** (from Deferred). Rationale:

- **CPIC Level A actionable guidance exists** for isoniazid + NAT2 (CPIC Tuberculosis / Isoniazid guideline). This satisfies the Tier 1 criterion "strong eQTL signal **and** CPIC Level A actionable evidence" from [`docs/validation-tiers.md`](validation-tiers.md) §Tier 1.
- **Physiology-side prerequisites resolved** by Sisyphus PR #32 (v0.3.2, 2026-05-06): NAT2 is in `sisyphus.predict.phenotype.PHENOTYPE_SCALES` and `data/physiology/reference_man.yaml`. The back-solve cancellation fix (also in PR #32) means non-transporter phenotype effects now propagate through to PK.
- **N(SA) expected substantially > N(PM SLCO1B1) on the same holdout.** NAT2 slow-acetylator allele frequency by 1000G super-population (literature): AFR ~0.30, AMR ~0.50, EAS ~0.10–0.15, EUR ~0.50, SAS ~0.50. Expected SA count in the 500-individual stratified holdout (132 AFR + 69 AMR + 101 EAS + 100 EUR + 98 SAS) is approximately **180–200**, vs the 3 PM SLCO1B1 in the same holdout. Tier 1 validation should therefore be substantially more statistically robust than SLCO1B1 was, and the sparse-evidence caveat that motivated the v0.3 PATH 3 band-widening will likely not apply with the same force.

Tier 1 it is. If a Phase 2 literature finding or implementation issue surfaces a reason to demote to Tier 2, that becomes a `tier-change:` per [`docs/commit-discipline.md`](commit-discipline.md) §4.

### 1.2 NAT2 SNP set + CPIC haplotype taxonomy

The five SNPs below are the **minimum sufficient set** for CPIC-standard NAT2 phenotype assignment (Slow / Intermediate / Rapid Acetylator). All five are well-genotyped in 1000 Genomes phase 3 short-read WGS.

| dbSNP rs | NAT2 position | functional impact (CPIC standard) | tags haplotypes |
|----------|---------------|------------------------------------|----------------|
| rs1801279 | c.191G>A      | non-synonymous, slow-function     | *14A, *14B    |
| rs1041983 | c.282C>T      | synonymous (linkage marker)       | *6A, *7B, *13 |
| rs1801280 | c.341T>C      | non-synonymous, slow-function     | *5A, *5B, *5C |
| rs1799930 | c.590G>A      | non-synonymous, slow-function     | *6A, *6B      |
| rs1799931 | c.857G>A      | non-synonymous, slow-function     | *7A, *7B      |

CPIC haplotype function classifications used in this protocol (locked):

| haplotype | required variants (all ALT)                   | function |
|-----------|------------------------------------------------|----------|
| `*4`      | none (all REF at the 5 SNPs above)            | **rapid** (reference) |
| `*5A`     | rs1801280                                      | slow     |
| `*5B`     | rs1801280 + rs1799929 (c.481C>T)              | slow     |
| `*5C`     | rs1801280 + rs1208 (c.803A>G)                 | slow     |
| `*6A`     | rs1041983 + rs1799930                          | slow     |
| `*6B`     | rs1799930 (without rs1041983)                  | slow     |
| `*7A`     | rs1799931                                      | slow     |
| `*7B`     | rs1041983 + rs1799931                          | slow     |
| `*14A`    | rs1801279                                      | slow     |
| `*14B`    | rs1801279 + rs1041983                          | slow     |

**Simplification for v0.4 phenotype call:** all "slow" haplotypes collapse to function = 0; `*4` (and any haplotype with no slow-variant ALT alleles) = function = 1. The collapsed call is sufficient for SA/IA/RA classification, which is what CPIC dosing guidance is keyed on. Haplotype *sub*-designations (*5A vs *5B vs *5C etc.) are NOT distinguished — only the slow-vs-rapid call per allele. This simplification is locked.

`*5B` and `*5C` require additional SNPs (rs1799929 and rs1208) beyond the minimum five. For v0.4, an individual carrying rs1801280 ALT (without specific sub-typing) is called as carrying a slow allele regardless of *5A/B/C distinction. This is conservative (assigns slow function to all rs1801280-ALT haplotypes) and matches CPIC's collapsed slow-function category.

`*12` (rs1208 ALT only, rapid function) and `*13` (rs1041983 ALT only, rapid function) are NOT separately handled. An individual whose only ALT call is rs1041983 will be assigned the conservative "slow" call (since rs1041983 is the *6/*7 sub linkage marker), which **may misclassify rare *13 carriers as slow**. This misclassification rate is expected to be low (*13 is rare in most populations) and is disclosed here. A v0.5+ refinement could add full *11/*12/*13 distinction.

### 1.3 Phenotype call algorithm

Per-individual: extract genotype at the 5 SNPs from the 1000G phase 3 VCF. For each allele (using phased data), determine whether any slow-function SNP carries the ALT. Then:

- `slow_count = number of alleles with slow function (0, 1, or 2)`
- Phenotype:
  - `slow_count == 2` → **Slow Acetylator (SA)**
  - `slow_count == 1` → **Intermediate Acetylator (IA)**
  - `slow_count == 0` → **Rapid Acetylator (RA)**

This is the CPIC-standard collapsing. Mirrors `genoadme.pgx.phenotype.call_phenotype`'s existing SLCO1B1 EM/IM/PM pattern.

**Phasing assumption:** 1000G phase 3 provides phased haplotypes (the panel was generated with statistical phasing). This is the same assumption already used for SLCO1B1. The known limitations of statistical phasing (rare-variant phasing error, recombination hotspots) apply but are disclosed in [`docs/limitations.md`](limitations.md) §4.

### 1.4 Validation criteria template (shape locked, reference values TBD)

The v0.4 Tier 1 criteria for NAT2/isoniazid follow the same three-criterion structure as SLCO1B1/pravastatin:

| criterion | locked at protocol time | TBD post-Phase-2 search |
|-----------|--------------------------|-------------------------|
| Population AAFE (AUC) ≤ X | gate type: AAFE on population-mean AUC against clinical reference | X (default 2.0 by analogy with SLCO1B1; revisit only if Phase 2 finds principled reason) |
| SA/RA AUC ratio in [Y, Z] | gate type: direct phenotype-effect magnitude (extremes-vs-extremes, analogous to SLCO1B1 PM/EM) | [Y, Z] band from Phase 2 meta-analysis of clinical SA-vs-RA AUC ratios |
| SA/RA Cmax (or t½) ratio ≥ W | gate type: direction-only floor | W from Phase 2 |

**Locked decisions:**

- Population-level AAFE on AUC, not Cmax (mirrors SLCO1B1 design — AUC integrates the acetylator-rate effect; Cmax is less divergent).
- Phenotype-effect ratios computed on **SA/RA extremes**, not SA-vs-mixed-population. IA is reported as a secondary metric (SA/IA, IA/RA ratios) but not gated.
- Lower-bound floor (criterion 3) keyed on Cmax OR t½ — the choice between them is deferred to Phase 2 because (a) different reference studies report different parameters, and (b) for isoniazid t½ is the more clinically-cited acetylator-effect proxy.

**NOT locked at protocol time:**

- Specific numerical values of Y, Z, W, X. These come from Phase 2 literature search + meta-analysis under the §1.5 search protocol. Direct-from-citation values without independent verification are explicitly prohibited (v0.3 PATH 3 lesson).
- Dose-stratification of criteria. Isoniazid is dosed at varying mg/kg in clinical studies (300 mg fixed dose, 5 mg/kg, 10 mg/kg). The Phase 2 search will determine whether to gate at a specific dose or pool across doses.

The full set of queries actually run is captured verbatim in §3.

**Inclusion criteria (locked):**

A study is **included** if:

1. **Original research** (not a review or prior meta-analysis). Reviews used only to discover candidate primary studies via reference lists.
2. **Healthy adult human volunteers** OR **TB patients on isoniazid monotherapy** (or fixed-dose combinations where isoniazid PK is reported separately). Mixed-drug protocols where isoniazid PK cannot be isolated are excluded.
3. **NAT2 genotyped or phenotyped** at SA/RA level minimum (intermediate IA reporting preferred but not required).
4. **Isoniazid PK reported** as group mean ± SD (or SE, IQR with N) for at least SA and RA groups. Studies reporting only pooled t½ ranges without group structure are excluded.
5. **N ≥ 4 per relevant phenotype group** (SA and RA). N ≥ 4 IA is a bonus (not required).
6. **Single-dose primary**, **multiple-dose / steady-state retained as supplementary**. Reported separately. Unlike SLCO1B1/pravastatin (v0.3 protocol §1.3 #3 excluded multi-dose entirely), multi-dose steady-state for isoniazid is clinically more relevant (isoniazid is dosed daily for months in TB treatment). But: NAT2 phenotype effect should be observable in single-dose, and single-dose is methodologically cleaner. Both extracted; primary pool uses single-dose only.

A study is **excluded** if any of:

- Pediatric, pregnant, or hepatic/renal-impaired cohorts.
- Isoniazid combined with rifampin, pyrazinamide, ethambutol, or other CYP-modulating drug where isoniazid PK is not reported in a separate cohort.
- N(SA) or N(RA) < 4.
- Acetylator phenotype determined from a single SNP that does not capture the population's haplotype diversity (this risks misclassifying *6/*7/*14 carriers as RA).

### 1.7 Analysis method

**Primary estimate: pooled log-ratio random-effects on SA/RA AUC ratio.** Same DerSimonian-Laird estimator as v0.3 (`docs/v0.3-meta-analysis.md` §1.5). Pool single-dose studies only; report multi-dose as a separate supplementary pool.

**Heterogeneity:** I² and Cochran's Q with df.

**Sensitivity analyses (pre-specified):**

1. High-confidence subset (`extraction_confidence == "high"`).
2. Leave-one-out.
3. Dose-stratified: restrict to commonly-cited 300 mg adult dose if multiple doses are represented.
4. NAT2-calling-method stratified: separate `genotype_haplotype`-called cohorts from `phenotype_caffeine`-called cohorts (the latter is a different operational definition of "slow acetylator").

**Decision rules (pre-specified, applied once to primary pooled estimate):**

- **D-A:** pooled CI extends below 1.5 or above 6.0 → potentially incompatible with the SLCO1B1-analog band shape; surface to user for spec-shape revision before locking criteria values.
- **D-B:** pooled CI entirely between 1.5 and 6.0 → set SA/RA AUC ratio band as `[pooled CI lower − margin, pooled CI upper + margin]` where margin = 0.5× CI half-width, with margin floor 0.3 and margin ceiling 1.5. The margin accommodates model-prediction noise relative to the empirical estimate.
- **D-C:** pooled estimate point but no usable CI (single primary study, no SD reported) → fall back to SLCO1B1-style band derivation: anchor band on the empirical CI's [lower, upper] if available, else surface to user.

These rules are applied **before** the validation run on the GenoADME holdout. The band is locked from this analysis, then validation proceeds.

### 1.8 Post-lock amendments

*None at time of locking.* If any change is needed during execution (e.g. discovery that the 1000G phase 3 VCF lacks one of the five SNPs in our marker set), it is appended here with date, reason, and pre-existing-evidence reference, and a corresponding `audit:` commit is made.

### 1.9 What this protocol is NOT

Same disclaimers as `docs/v0.3-meta-analysis.md` §1.6:

- Not a formal Cochrane meta-analysis. No PROSPERO registration.
- No grey-literature or non-English search.
- Full-text access best-effort; abstracts-only counted as `medium` confidence.
- Single-rater extraction.
- No formal risk-of-bias scoring.

These limitations are accepted because the goal is to inform v0.4 closing, not publish a standalone meta-analysis. Honest disclosure in the report is the rigor substitute.

Two NAT2-specific risks worth pre-disclosing:

- **NAT2 *11/*12/*13 (rapid alleles) not separately distinguished.** Conservative mis-classification of these as slow is possible per §1.2.
- **NAT2 calling methodology heterogeneity.** Older studies (pre-2000) often used caffeine-metabolic-ratio phenotyping rather than genotyping. Pooling across calling-method types is a potential heterogeneity source; the §1.7 stratified sensitivity analysis exposes this.

### 2.2 Studies included / excluded — PRISMA-lite flow

| study | dose / design | N(SA) | N(RA) | direct-observation? | inclusion verdict |

**Net primary inclusion:** **one study** (Kinzig-Schippers 2005), again echoing the v0.3 SLCO1B1 sparse-evidence pattern. Reasons primary inclusion is so thin:

- Most NAT2/INH PK papers report population PK *models* (popPK), not per-phenotype group means with SDs.
- TB-patient studies often report EBA (early bactericidal activity) endpoints rather than per-phenotype AUC.
- Healthy-volunteer dose-escalation studies (like Kubota 2007) often only enroll a subset of phenotypes.
- Older papers (Parkin 1997, Donald 2007) require full-text extraction beyond abstract scope.

Better extraction would require systematic full-text access (CPIC supplement PDFs, paywalled journals). The honest scope of this v0.4 Phase 2 is what's accessible via abstract-level reading.

### 2.3 Primary estimate

**Single-study primary estimate from Kinzig-Schippers 2005:**

- SA (NAT2*5/*5, *5/*6, *6/*6, N=13): AUC/dose = 0.091 h/L (range 0.067–0.132)
- RA (NAT2*4/*4, N=3): AUC/dose = 0.031 h/L (range 0.027–0.037)
- **SA/RA AUC ratio = 2.94**
- SDs approximated via range/4 (Hozo et al. 2005, symmetric assumption): SA SD ≈ 0.0163, RA SD ≈ 0.0025
- Delta-method variance on log-scale: SE(log ratio) ≈ 0.068
- **95% CI for SA/RA AUC ratio: [2.57, 3.35]**

Honest framing of the CI: this is a single-study CI with SDs estimated from ranges. The true SDs may be larger (range/4 underestimates SD for non-uniform distributions), which would widen the CI. The narrow [2.57, 3.35] should be treated as a *lower bound* on the true CI width.

**Supplementary signals** (not in primary pool):

- Surarak 2024 aggregate (41-study review): SA/RA AUC = 40.38 / 19.46 = 2.08. **Below** the Kinzig-Schippers CI lower bound 2.57.
- Seng 2015 popPK Asians: SA/RA AUC implied via CL = 65/8 = **8.12**. **Above** the Kinzig-Schippers CI upper bound 3.35 by a wide margin.
- Gumbo 2007 review t½ midpoint proxy: 3.3h / 1.35h = 2.44. Inside Kinzig-Schippers CI.

The Kinzig-Schippers single-study CI [2.57, 3.35] does NOT capture the heterogeneity signaled by Surarak (lower) and Seng (much higher). This is a real limitation of single-study pooling.

### 2.4 Sensitivity analyses

Pre-spec'd sensitivity #1 (high-confidence subset): Kinzig-Schippers is extraction_confidence=`medium` (SDs from ranges). Restricting to `high` confidence yields no primary studies → cannot apply.

Pre-spec'd sensitivity #2 (leave-one-out): N=1 primary study → cannot apply.

Pre-spec'd sensitivity #3 (dose-stratified at 300 mg): Kinzig-Schippers is at 100 mg. Cannot apply at 300 mg.

Pre-spec'd sensitivity #4 (NAT2 calling-method stratified): Kinzig-Schippers uses `genotype_haplotype`. Seng 2015 uses `popPK_model` (excluded from primary). The protocol-disclosed methodological heterogeneity (genotype vs popPK vs phenotype) is real and reflected by the wide range of supplementary signals (2.08, 8.12) vs primary CI (2.57–3.35).

**Sensitivity finding:** the primary pool's narrow CI under-reports inter-study + inter-population heterogeneity. The "true" SA/RA AUC ratio for the world population is likely in a wider range than [2.57, 3.35].

### 2.5 Decision rule applied + locked validation criteria

Applying protocol §1.7 decision rule:

- Primary CI [2.57, 3.35] is entirely in [1.5, 6.0] → **D-B fires.**
- Margin = max(0.3, min(1.5, 0.5 × CI half-width)) = max(0.3, min(1.5, 0.5 × 0.39)) = max(0.3, 0.20) = **0.30 (floor)**
- Band = [CI lower − margin, CI upper + margin] = [2.57 − 0.30, 3.35 + 0.30] = [2.27, 3.65]
- Single-decimal rounding: **[2.3, 3.7]**

**Locked Tier 1 NAT2/isoniazid validation criteria** (mirror SLCO1B1 three-criterion structure):

| criterion | locked value | source |
|-----------|--------------|--------|
| Population AAFE (AUC) ≤ | **2.0** | Default per protocol §1.4 (mirror SLCO1B1). Reference AUC anchor value remains TBD during Phase 3 implementation; candidate anchors include (a) Kinzig-Schippers RA-arm extrapolated to 300mg (~9.3 mg·h/L if linear, ~8.1 mg·h/L if sub-linear per Kubota 2007), or (b) Surarak 2024 aggregate RA mean 19.46 mg·h/L. Decision deferred to Phase 3. |
| SA/RA AUC ratio in | **[2.3, 3.7]** | D-B formula output |
| SA/RA Cmax ratio ≥ | **1.3** | Direction floor; mirror SLCO1B1. Empirically Kinzig-Schippers SA/RA Cmax = 2.10/1.20 = 1.75, comfortably above 1.3. |

**Pre-disclosed sparse-evidence caveat:** the SA/RA AUC ratio band [2.3, 3.7] is anchored on a SINGLE primary direct-observation study with N(RA)=3 (marginally below the pre-spec N≥4 threshold; the same sparse-evidence shape as SLCO1B1 v0.3 PATH 3, where Niemi 2006 men-stratum N(CC)=3 was the anchor). The supplementary heterogeneity signal (Surarak aggregate 2.08 below band, Seng popPK 8.12 way above band) suggests the band may need post-implementation revision if the model output lands outside but inside a wider empirical interval. This is the v0.3 PATH 3 pattern; the §1.7 D-B formula was applied as locked, and any band revision becomes a `tier-change:` per [`docs/commit-discipline.md`](commit-discipline.md) §4 with new evidence cited.
